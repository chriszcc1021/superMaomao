# Phase 7：美术与视觉基线清单

目标：不改核心玩法，先把项目从“程序占位版”推进到“可展示、可试玩、风格统一”的第一版正式视觉。

## 美术方向

- 风格：温暖 Q 版、轻手绘、清晰轮廓、低复杂度细节。
- 视角：营地使用等距视角，战斗使用俯视/轻俯视视角。
- 气质：猫咪经营 + 远征冒险，避免过暗、过写实、过复杂。
- 原则：先高频可见资源，后低频装饰；先静态正式稿，后动画精修。

## 资产规格

- 营地建筑：`128x128` PNG，透明背景。
- 营地猫咪：`64x64` PNG，透明背景。
- 战斗猫咪：`64x64` PNG，透明背景。
- 敌人：小怪 `48x48`，精英/Boss `64x64` 或 `96x96`，透明背景。
- 卡牌图标：`96x96` PNG，透明背景。
- UI 图标：`32x32` 或 `48x48` PNG，透明背景。
- 战斗地面 tile：`128x128` 可循环 PNG。
- UI 面板皮肤：优先用 Godot StyleBox/TextureRect 接入，图片只做必要底纹和边框。

建议目录：

```text
assets/art/
├── camp/buildings/
├── camp/cats/
├── battle/player/
├── battle/enemies/
├── battle/tiles/
├── cards/icons/
├── ui/icons/
└── ui/skins/
```

## AI 生成策略

可以直接生成：

- 建筑单张 PNG。
- 猫咪/敌人静态角色图。
- 卡牌图标。
- UI 小图标。
- 战斗地面纹理。
- 面板底纹和装饰边框。

第一轮先不生成：

- 复杂逐帧动画。
- 每张卡的完整插画。
- 大量剧情立绘。
- 过细的建筑升级多阶段图。

原因：第一轮重点是统一视觉和接入路径，动画和大量插画容易拖慢验收。
